API Reference

Clean REST endpoints.
Predictable responses.

Every endpoint accepts and returns JSON. Authentication is via a bearer token in the Authorization header. All amounts are returned as decimal strings to avoid floating-point errors.

BIP-39 Key Derivation

Your agent owns
its own keys.

Every wallet is generated from a standard BIP-39 mnemonic phrase â€” 12 or 24 words that deterministically derive every address your agent will ever use. Back up the mnemonic and you can reconstruct the entire wallet on any standard BIP-32 client, independent of Purple Flea.

Chain Derivation Path Address Format
Ethereum / BNB / Polygon m/44'/60'/0'/0/0 0x hex (EIP-55 checksum)
Bitcoin m/84'/0'/0'/0/0 bc1q... (native SegWit)
Solana m/44'/501'/0'/0' Base58 encoded pubkey
TRON m/44'/195'/0'/0/0 T... (Base58Check, 0x41)
🔑

Deterministic address generation

A single 12-word mnemonic generates every address across all six supported chains. The same seed, the same derivation path, the same address â€” every time. Your agent's identity is fully portable and self-sovereign.

Private keys are encrypted at rest using AES-256-GCM. The mnemonic is only ever returned once, at wallet creation. Store it securely â€” Purple Flea cannot recover it.

DEX Swap Routing

Best-rate aggregation
across every chain.

When your agent calls POST /wallet/swap, the routing engine queries multiple DEX aggregators simultaneously and executes on whichever route offers the best post-fee output. Slippage tolerance, price impact warnings, and minimum received amounts are all configurable per request.

🔄

Multi-DEX aggregation

Routes are evaluated across Uniswap v2/v3 and 1inch on Ethereum; PancakeSwap on BNB Chain; Jupiter on Solana; and native DEXs on Polygon and Tron. Your agent always gets the best available rate.

⚡

Slippage control

Set slippage_bps per swap (default: 50 bps = 0.5%). The transaction will revert on-chain if the final output falls below your minimum. Protects your agent from sandwich attacks and volatile price moves.

📈

Price impact warnings

For large swaps relative to pool liquidity, the API returns a price_impact_pct field and a warning flag when impact exceeds 1%. Your agent can decide whether to split the order or abort.
