EVM Compatibility

Same address as Ethereum. Different chain ID.

BNB Chain uses the same elliptic curve and address format as Ethereum. Your agent's BSC address is identical to its Ethereum address â€” the only difference is which network you submit transactions to. Purple Flea uses chain ID 56 (BSC mainnet) when constructing BNB Chain transactions.

The BIP-44 derivation path for BNB Chain follows Ethereum's coin type: m/44'/60'/0'/0/0. This means a single mnemonic produces the same address on both Ethereum and BSC. Purple Flea derives this path internally and presents it as your BSC address in the API response.

PancakeSwap Integration

Best-rate swaps on BSC.
Slippage you control.

PancakeSwap v3 is the leading DEX on BNB Chain. Purple Flea routes swaps through PancakeSwap's concentrated liquidity pools for optimal execution.

pancakeswap_swap.json
// Swap BNB for USDT via PancakeSwap POST /wallet/swap { "chain": "bsc", "from_token": "bnb", "to_token": "usdt", "amount": "0.5", "slippage_bps": 50 } // Response { "tx_hash": "0xabc123...", "from_amount": "0.5 BNB", "to_amount": "147.82 USDT", "price_impact": "0.03%", "fee_bnb": "0.00021", "dex": "pancakeswap_v3" }

PancakeSwap v3 introduced concentrated liquidity, letting liquidity providers focus capital in narrow price ranges. For traders, this means tighter spreads and less price impact â€” especially on major pairs like BNB/USDT.

Purple Flea's swap API accepts a slippage tolerance in basis points (50 bps = 0.5%). If the on-chain price moves beyond your tolerance between submission and execution, the transaction reverts automatically. Your agent never gets a worse fill than specified.

CAKE is PancakeSwap's native token. Agents that hold and stake CAKE receive reduced trading fees and participate in governance. The Purple Flea wallet API supports CAKE transfers and PancakeSwap interactions for agents that want to optimize trading costs over time.
