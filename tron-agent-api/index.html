Derivation Path

BIP-39 HD wallet,
TRON address format.

Purple Flea derives your TRON address deterministically from your BIP-39 mnemonic using the standard TRON coin type.

tron_derivation.txt
# TRON BIP-44 derivation path # Coin type 195 = TRON (registered in SLIP-44) Path: m/44'/195'/0'/0/0 # Address encoding steps: 1. Derive secp256k1 key pair at path 2. Keccak-256 hash of uncompressed public key 3. Take last 20 bytes (EVM-style address) 4. Prepend 0x41 byte (TRON mainnet prefix) 5. Double-SHA256 of (0x41 + address) â†’ 4-byte checksum 6. base58check(0x41 + address + checksum) # Result: address starting with 'T' # Example: TQn9Y2khEsLJW1ChVWFMSMeRDow5KcbLSE

TRON uses the same underlying elliptic curve (secp256k1) and key derivation as Ethereum, but encodes addresses differently. The last 20 bytes of the keccak hash of the public key form the raw address â€” identical to the Ethereum EVM address â€” but TRON prepends a 0x41 network byte and encodes the whole thing as base58check rather than hex.

The result is always a 34-character string beginning with the letter T. Your agent's TRON address is deterministic: given the same BIP-39 mnemonic, you always get the same T-address. There is no registration required and no on-chain account creation step.

Purple Flea manages all of this derivation internally. You register an agent, receive a mnemonic, and immediately have a working TRON address. The address is yours â€” you can import the mnemonic into any standard TRON wallet such as TronLink to verify control.

Agent-to-Agent Payments

Pay per request.
At $0.001 per tx, it makes sense.

At $0.001 per TRON transaction, agents can pay each other per individual API call rather than subscribing to monthly plans. This unlocks new agent-native business models.

Imagine an orchestrator agent that needs a specialized analysis. Instead of pre-purchasing a subscription, it sends $0.05 in TRC-20 USDT to a worker agent's TRON address, then polls for the result. The worker verifies payment on-chain before processing. Total overhead: $0.001 for the TRON transaction fee â€” essentially free.

This pay-per-request pattern works because TRON's economics are different from every other chain. On Ethereum, paying $1â€“5 to transfer $0.05 is absurd. On TRON, the transaction fee is smaller than the rounding error on your payment.

Purple Flea's TRON API supports this pattern natively. Your orchestrator agent can call POST /wallet/send with a small USDT amount before every downstream API call. Worker agents verify receipt via GET /wallet/transactions and deliver results only to paying callers.

The result is an agent economy where computation, data, and intelligence are priced at their true marginal cost â€” with no payment processor, no subscription overhead, and no human in the loop.

Bandwidth and Energy

TRON's resource model,
handled automatically.

TRON uses a unique resource model instead of variable gas fees. Purple Flea manages this for you.

📶

Bandwidth

Every TRON account gets 600 free bandwidth points per day. Bandwidth is consumed by TRX transfers. Exceeding the free limit burns TRX at a tiny rate. For high-volume agents, bandwidth can be obtained by freezing TRX.

⚡

Energy

TRC-20 token transfers require Energy, which covers smart contract execution. Energy is consumed when calling the USDT contract's transfer() function. Like bandwidth, it can be obtained by freezing TRX or paid in TRX directly.

⚙️

Auto-managed by Purple Flea

You never interact with bandwidth or energy directly. Purple Flea maintains a TRX reserve in your agent wallet and uses it to cover resource costs automatically. The $0.001 fee you see is the all-in cost of a TRC-20 transfer.
