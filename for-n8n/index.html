Installation

One npm install.

Install the n8n-nodes-purpleflea community node directly from your n8n instance settings, or via npm in a self-hosted environment. Alternatively, use the built-in HTTP Request node with the Purple Flea REST API â€” no custom node required.

AI Agent Workflows

Common patterns for
autonomous n8n agents.

n8n's AI Agent node (powered by LangChain under the hood) can call Purple Flea nodes as tools. These three workflow patterns cover the most common autonomous finance use cases.

1

Conversational Trading Agent

A chat-triggered workflow where an AI agent interprets a user message, decides whether to open a trade, executes the position via the Trading node, and sends back a Slack or Telegram confirmation with entry price and position size. The agent handles ambiguous instructions like "go long BTC if it dips below 60k" without any extra logic code.

Chat Trigger → AI Agent → Purple Flea Trading (Open Position) → Slack / Telegram
2

Funding Rate Arbitrage Bot

A scheduled workflow that fires every hour, fetches funding rates for multiple perpetual markets via the Trading node's Get Price operation, compares current rates against a configurable threshold (e.g. 0.05%), and automatically opens a delta-neutral position when the funding rate exceeds the threshold. Positions are automatically closed when rates normalize. No server required â€” n8n handles the scheduling and state.

Cron (every hour) → Purple Flea Trading (Get Price) → IF (rate > 0.05%) → Purple Flea Trading (Open Position)
3

Domain Sniper Agent

An AI agent monitors incoming chat messages or RSS feeds for domain names being mentioned. When a domain is detected, the Domains node checks availability across 500+ TLDs. If the domain is available and under a configurable price threshold, the agent registers it immediately using the agent's crypto wallet, then logs the registration to a Google Sheet or Airtable for review.

Chat / RSS Trigger → AI Agent (extract domain) → Purple Flea Domains (Search) → IF (available) → Purple Flea Domains (Register)

Credentials Setup

Authenticate once.
Use everywhere.

Purple Flea uses API key authentication. Create a credential in n8n and reference it across all Purple Flea nodes and HTTP Request nodes in your workspace.

01

Get your API key

Sign up at wallet.purpleflea.com. Your API key starts with pf_sk_. Each key is tied to a BIP-39 HD wallet that holds your agent's funds across all chains.

02

Create n8n credential

In n8n, go to Credentials → Add Credential → Purple Flea API. Enter your API key in the apiKey field. Select production or testnet from the Base URL dropdown. n8n encrypts and stores the credential securely.

03

Reference in nodes

Every Purple Flea node has a Credentials field at the top. Select the credential you just created. For HTTP Request nodes, use Header Auth with name X-API-Key.

04

Base URL options

Production: https://api.purpleflea.com/v1 â€” Testnet: https://testnet-api.purpleflea.com/v1. Use testnet for workflow development and testing without real funds.
