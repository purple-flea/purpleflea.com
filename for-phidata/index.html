Framework

What is PhiData / Agno?

PhiData rebranded to Agno in 2024. It's a lightweight Python framework purpose-built for production AI agents. Where LangChain abstracts everything, Agno stays out of your way: you write plain Python, decorate with @tool, hand the list to Agent(), and the framework handles tool-calling, memory, knowledge retrieval, and streaming end-to-end.

🔧

Tool-first design

Any Python function decorated with @tool becomes a callable the model can invoke. Docstrings become descriptions automatically.

🧠

Built-in memory

Agents persist conversation history, user facts, and session state in SQLite or Postgres. No boilerplate — just pass memory=AgentMemory().

📚

Knowledge bases

Connect PDFs, URLs, or databases as searchable knowledge. Your agent retrieves relevant context before answering, no prompt engineering required.

Setup

Install in 30 seconds.

Install agno (the new package name) plus requests for the Purple Flea REST calls. If you are on an older project using the original package name, phidata works identically — the API is the same.

$ pip install agno requests
$ pip install phidata requests  — older projects
quickstart.py 
from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.tools import tool
import requests

PURPLEFLEA_KEY = "sk_live_..."
BASE = "https://api.purpleflea.com"

@tool
def get_balance() -> str:
    """Get the current Purple Flea account balance in USD."""
    r = requests.get(f"{BASE}/api/v1/wallet/balance",
        headers={"Authorization": f"Bearer {PURPLEFLEA_KEY}"})
    data = r.json()
    return f"Balance: ${data.get('usd_balance', 0):.2f}"

agent = Agent(
    model=Claude(id="claude-opus-4-6"),
    tools=[get_balance],
    show_tool_calls=True,
)

agent.print_response("What is my current balance?")
Tool Definitions

Purple Flea as Agno @tool decorators.

The @tool decorator is the Agno way. The model reads the function signature and docstring to decide when and how to call each tool. No schema files, no config — just annotated Python.

purpleflea_tools.py 
from agno.tools import tool
import requests

PURPLEFLEA_KEY = "sk_live_..."
BASE = "https://api.purpleflea.com"

def _h():
    return {"Authorization": f"Bearer {PURPLEFLEA_KEY}"}

# ── Casino ──────────────────────────────────────────────────────

@tool
def place_casino_bet(game: str, amount: float, side: str) -> str:
    """Place a bet on the Purple Flea casino.
    Games: coin_flip, dice, roulette, crash.
    side: heads/tails for coin_flip; over/under for dice;
          red/black/number for roulette; a float cashout multiplier for crash."""
    endpoint = {
        "coin_flip": "/api/v1/flip",
        "dice":      "/api/v1/dice",
        "roulette":  "/api/v1/roulette",
        "crash":     "/api/v1/crash",
    }.get(game, "/api/v1/flip")
    r = requests.post(BASE + endpoint, headers=_h(),
        json={"amount": amount, "side": side})
    d = r.json()
    result = "WIN" if d.get("won") else "LOSS"
    return f"{result} | payout=${d.get('payout',0):.2f} | proof={d.get('proof','')}"

# ── Trading ─────────────────────────────────────────────────────

@tool
def open_trade(coin: str, side: str, size_usd: float, leverage: int = 5) -> str:
    """Open a perpetual futures position via Purple Flea Trading API.
    coin: BTC, ETH, SOL, etc. side: long or short. leverage: 1-50."""
    r = requests.post(BASE + "/v1/trading/open", headers=_h(),
        json={"market": f"{coin.upper()}-PERP", "side": side,
             "size": size_usd, "leverage": leverage})
    d = r.json()
    return f"Opened {side} {coin.upper()}-PERP ${size_usd} x{leverage} | id={d.get('position_id')}"

@tool
def close_trade(position_id: str) -> str:
    """Close an open perpetual futures position by position ID."""
    r = requests.post(BASE + "/v1/trading/close", headers=_h(),
        json={"position_id": position_id})
    d = r.json()
    pnl = d.get("pnl", 0)
    return f"Closed {position_id} | PnL={'+' if pnl >= 0 else ''}{pnl:.2f} USD"

@tool
def get_market_price(coin: str) -> str:
    """Get the current mark price for a crypto perpetual market."""
    r = requests.get(BASE + "/v1/trading/price", headers=_h(),
        params={"market": f"{coin.upper()}-PERP"})
    d = r.json()
    return f"{coin.upper()} mark price: ${d.get('price', 0):,.2f}"

# ── Wallet ──────────────────────────────────────────────────────

@tool
def get_wallet_balance(chain: str = "ethereum") -> str:
    """Get wallet token balances. chain: ethereum, bitcoin, solana, tron, bnb, polygon."""
    r = requests.get(BASE + "/api/v1/wallet/balance", headers=_h(),
        params={"chain": chain})
    return str(r.json())
Complete Example

Full agent: $100 portfolio
across casino + trading.

A working 70+ line agent that manages a starting bankroll of $100 using Purple Flea tools. It checks prices, places trades with stop-loss logic, tries a casino bet, and reports its final position — all autonomously.

portfolio_agent.py 
"""
Purple Flea x Agno — autonomous $100 portfolio manager.
Manages casino bets and perpetual trades within a set bankroll.
"""

from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.tools import tool
from agno.memory.agent import AgentMemory
from agno.storage.agent.sqlite import SqlAgentStorage
import requests, json

PURPLEFLEA_KEY = "sk_live_..."
BASE = "https://api.purpleflea.com"
STARTING_BANKROLL = 100.0
MAX_BET_FRACTION  = 0.05   # never risk more than 5 % of bankroll per bet
MAX_TRADE_SIZE    = 20.0   # max notional per trade in USD

def _h():
    return {"Authorization": f"Bearer {PURPLEFLEA_KEY}"}

@tool
def get_account_summary() -> str:
    """Return current USD balance, open positions, and recent P&L summary."""
    bal = requests.get(BASE + "/api/v1/wallet/balance", headers=_h()).json()
    pos = requests.get(BASE + "/v1/trading/positions",  headers=_h()).json()
    return json.dumps({
        "usd_balance": bal.get("usd_balance", 0),
        "open_positions": pos.get("positions", []),
        "max_safe_bet": round(bal.get("usd_balance", 0) * MAX_BET_FRACTION, 2),
        "max_trade_size": MAX_TRADE_SIZE,
    })

@tool
def place_casino_bet(game: str, amount: float, side: str) -> str:
    """Place a provably fair casino bet. Games: coin_flip, dice, roulette, crash."""
    ep = {"coin_flip":"/api/v1/flip", "dice":"/api/v1/dice",
          "roulette":"/api/v1/roulette", "crash":"/api/v1/crash"}.get(game, "/api/v1/flip")
    r = requests.post(BASE + ep, headers=_h(),
                       json={"amount": amount, "side": side})
    d = r.json()
    return json.dumps({"outcome": "win" if d.get("won") else "loss",
                        "payout": d.get("payout",0), "proof": d.get("proof","")})

@tool
def open_trade(coin: str, side: str, size_usd: float, leverage: int = 5) -> str:
    """Open a perpetual futures position. coin: BTC/ETH/SOL. side: long/short."""
    r = requests.post(BASE + "/v1/trading/open", headers=_h(),
        json={"market": f"{coin.upper()}-PERP", "side": side,
             "size": size_usd, "leverage": leverage})
    d = r.json()
    return json.dumps({"position_id": d.get("position_id"),
                        "entry_price": d.get("entry_price"),
                        "market": f"{coin.upper()}-PERP", "side": side})

@tool
def close_trade(position_id: str) -> str:
    """Close an open perpetual futures position by its ID."""
    r = requests.post(BASE + "/v1/trading/close", headers=_h(),
                       json={"position_id": position_id})
    d = r.json()
    return json.dumps({"pnl": d.get("pnl",0), "closed_at": d.get("closed_at")})

@tool
def get_price(coin: str) -> str:
    """Get current mark price for a crypto perpetual (BTC, ETH, SOL, etc.)."""
    r = requests.get(BASE + "/v1/trading/price", headers=_h(),
                      params={"market": f"{coin.upper()}-PERP"})
    d = r.json()
    return f"{coin.upper()} = ${d.get('price', 0):,.2f}"

# ── Build the agent ─────────────────────────────────────────────

agent = Agent(
    model=Claude(id="claude-opus-4-6"),
    tools=[get_account_summary, place_casino_bet, open_trade, close_trade, get_price],
    memory=AgentMemory(),
    description="Autonomous portfolio manager with $100 starting bankroll",
    instructions=[
        "Always call get_account_summary first to know your current balance.",
        "Never risk more than 5% of your balance on a single casino bet.",
        "Cap each trade at $20 notional regardless of bankroll size.",
        "After each action, report the new balance and cumulative P&L.",
        "Verify casino bet proof hashes before reporting outcomes.",
    ],
    show_tool_calls=True,
    markdown=True,
)

agent.print_response(
    "Manage my $100 portfolio: check prices, open a small ETH long, "
    "place a coin flip bet, then summarise the portfolio state.",
    stream=True,
)
Persistent State

Agent memory: balance and
bet history across sessions.

Agno's AgentMemory + SqlAgentStorage lets the agent remember its balance, open positions, recent bet outcomes, and user preferences between conversations. No external database needed for development — just a local SQLite file.

memory_agent.py 
from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.memory.agent import AgentMemory
from agno.memory.db.sqlite import SqliteMemoryDb
from agno.storage.agent.sqlite import SqlAgentStorage

# Persistent memory — agent recalls balance, bets, and positions
memory = AgentMemory(
    db=SqliteMemoryDb(table_name="pf_memory", db_file="pf_agent.db"),
    create_user_memories=True,   # store facts about the user/account
    create_session_summary=True, # summarise each session for future recall
)

storage = SqlAgentStorage(
    table_name="pf_sessions",
    db_file="pf_agent.db",
)

agent = Agent(
    model=Claude(id="claude-opus-4-6"),
    tools=[get_account_summary, place_casino_bet, open_trade, close_trade, get_price],
    memory=memory,
    storage=storage,
    add_history_to_messages=True,
    num_history_responses=10,
    description="Portfolio agent with persistent memory of balance and recent bets",
    instructions=[
        "Remember the user's starting bankroll as $100.",
        "Track every bet outcome and running P&L in memory.",
        "Recall all open positions from previous sessions.",
        "If asked 'how did I do last session?', use stored summaries.",
    ],
    show_tool_calls=True,
)

# Session 1
agent.print_response("Flip a coin for $5. Remember the result.")

# Session 2 — agent remembers the bet without being told
agent.print_response("Did I win my last casino bet? What's my current P&L?")

Streaming + Structured Outputs

Real-time reasoning and
JSON-typed responses.

Pass stream=True to watch the agent think and call tools live. Define a Pydantic model as response_model and Agno forces the model to return structured JSON — ideal for downstream automation.

streaming_structured.py 
from agno.agent import Agent
from agno.models.anthropic import Claude
from pydantic import BaseModel
from typing import Literal, Optional

# ── Structured output schema ─────────────────────────────────────

class TradeAction(BaseModel):
    action:      Literal["bet", "trade", "hold"]
    coin:        Optional[str]  = None
    side:        Optional[str]  = None
    amount_usd:  float
    result:      str
    new_balance: float
    reasoning:   str

# ── Streaming agent with structured output ───────────────────────

agent = Agent(
    model=Claude(id="claude-opus-4-6"),
    tools=[get_account_summary, place_casino_bet, open_trade, get_price],
    response_model=TradeAction,
    description="Returns a structured action summary after every financial operation",
    show_tool_calls=True,
)

# Watch the agent reason in real time, then get typed output
response: TradeAction = agent.run(
    "Check ETH price, then open a $10 long with 3x leverage.",
    stream=True,
)

# response is a fully typed Pydantic model
print(f"Action  : {response.action}")
print(f"Result  : {response.result}")
print(f"Balance : ${response.new_balance:.2f}")
print(f"Reason  : {response.reasoning}")

# Typical output:
# Action  : trade
# Result  : Opened ETH-PERP long $10 x3 | id=pos_abc123
# Balance : 90.00
# Reason  : ETH is at a local support. 3x leverage keeps risk manageable.
