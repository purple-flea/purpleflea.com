Overview

Why Purple Flea is a Natural Fit for CrewAI

CrewAI's power comes from specialisation: each agent in a crew owns a domain of expertise and a focused toolkit. Purple Flea mirrors that philosophy exactly.

When you design a multi-agent financial system, you want agents that are narrowly scoped â€” a Portfolio Analyst that only reads prices, a Trader that only executes swaps, a Risk Manager that only monitors positions. crewai-purpleflea lets you hand each agent exactly the tools it needs, nothing more. The @tool decorator pattern maps cleanly to CrewAI's tool registration model: decorate a function, add a clear docstring so the LLM knows when to use it, and attach it to an agent. Done.

Under the hood, each Purple Flea tool function calls the same production-grade async API layer that powers the LangChain integration â€” with identical non-custodial wallet signing, Pydantic-validated inputs, and structured logging. The CrewAI wrapper adds three things on top: synchronous execution wrappers that play nicely with CrewAI's internal task runner, docstring-driven tool descriptions optimised for role-based prompting, and pre-built example agents you can drop into a crew immediately.

The result is a crypto-capable crew architecture where agents collaborate to handle complex multi-step on-chain workflows that would overwhelm a single agent â€” think researching token fundamentals, evaluating risk, executing a swap, then registering a brand domain to announce the new position, all orchestrated by a CrewAI manager without a single line of glue code.

Tool Definitions

Four @tool Functions, Ready to Import

crewai-purpleflea exports four functions already decorated with CrewAI's @tool decorator. Import them directly and attach to any Agent â€” no extra setup needed.

tools.py â€” how the tools are defined internally python 
from crewai.tools import tool
from crewai_purpleflea._client import PurpleFlecaClient

_client = PurpleFlecaClient()  # reads PURPLEFLEA_API_KEY from env

# â”€â”€ Wallet tool â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
@tool("crypto_wallet")
def wallet_tool(action: str, chain: str = "ethereum", token: str = "",
                 amount: str = "", to_address: str = "") -> str:
    """Manage a non-custodial crypto wallet. Actions: get_balance,
    get_portfolio, transfer, get_address. Supports Ethereum, Arbitrum,
    Polygon, Base, Solana, Bitcoin. Returns JSON with balances and
    transaction details. Use for all wallet and balance queries."""
    return _client.wallet(action=action, chain=chain, token=token,
                          amount=amount, to_address=to_address)

# â”€â”€ Trading tool â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
@tool("onchain_trading")
def trading_tool(action: str, market: str = "", side: str = "",
                  size_usd: str = "", from_token: str = "",
                  to_token: str = "", leverage: int = 1,
                  chain: str = "arbitrum") -> str:
    """Execute on-chain trades and swaps. Actions: swap, open_position,
    close_position, get_positions, get_price. Supports spot swaps via
    DEX aggregators and leveraged perpetuals. Returns execution details
    including tx hash, fill price, and gas cost."""
    return _client.trading(action=action, market=market, side=side,
                           size_usd=size_usd, from_token=from_token,
                           to_token=to_token, leverage=leverage, chain=chain)

# â”€â”€ Casino tool â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
@tool("provably_fair_casino")
def casino_tool(game: str, bet_usd: float = 1.0, choice: str = "",
                 target: str = "", target_value: int = 50) -> str:
    """Play provably-fair on-chain casino games. Games: coin_flip,
    dice_roll, roulette, hi_lo, crash. All outcomes use VRF randomness
    verifiable on-chain. Returns result, payout, and VRF proof.
    Use only with explicit user consent and appropriate bet sizing."""
    return _client.casino(game=game, bet_usd=bet_usd, choice=choice,
                          target=target, target_value=target_value)

# â”€â”€ Domains tool â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
@tool("web3_domains")
def domains_tool(action: str, names: list = None, name: str = "",
                  duration_years: int = 1, records: dict = None) -> str:
    """Register and manage Web3 domains across ENS, Unstoppable Domains,
    and Handshake. Actions: check_availability, register, renew, resolve,
    set_records, list_owned. Returns availability, pricing, and tx details.
    Use for all domain registration and lookup tasks."""
    return _client.domains(action=action, names=names or [], name=name,
                           duration_years=duration_years, records=records or {})
Tip: The docstrings on each @tool function are what the LLM reads to decide when to invoke the tool. Keep them crisp, action-oriented, and accurate. The pre-written docstrings in crewai-purpleflea are tuned for maximum clarity with GPT-4o and Claude Opus.
Full Example

Complete Agent + Task + Crew Setup

This example builds a three-agent DeFi crew: a Portfolio Analyst that reads current holdings, a Trader that executes the best swap, and a Recorder that registers a Web3 domain to log the trade publicly.

defi_crew.py â€” full CrewAI setup with Purple Flea tools python 
from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI
from crewai_purpleflea import (
    wallet_tool,
    trading_tool,
    domains_tool,
    casino_tool,
)

llm = ChatOpenAI(model="gpt-4o", temperature=0)

# â”€â”€ Agent 1: Portfolio Analyst â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
portfolio_analyst = Agent(
    role="Senior DeFi Portfolio Analyst",
    goal="Analyse current wallet holdings and identify the best swap"
         " opportunity to maximise USDC yield on Arbitrum.",
    backstory=(
        "You are a seasoned on-chain analyst who has navigated multiple"
        " DeFi cycles. You check wallet balances, query live prices, and"
        " surface actionable trade recommendations with clear rationale."
    ),
    tools=[wallet_tool, trading_tool],
    llm=llm,
    verbose=True,
    allow_delegation=False,
)

# â”€â”€ Agent 2: On-Chain Trader â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
trader = Agent(
    role="Precision On-Chain Trader",
    goal="Execute the swap recommended by the analyst with minimal"
         " slippage, verify the outcome, and report the fill price.",
    backstory=(
        "You execute trades methodically. You always check the current"
        " price before trading, set a slippage tolerance appropriate"
        " to market conditions, and confirm the transaction on-chain."
    ),
    tools=[trading_tool, wallet_tool],
    llm=llm,
    verbose=True,
    allow_delegation=False,
)

# â”€â”€ Agent 3: Brand & Domain Registrar â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
domain_registrar = Agent(
    role="Web3 Brand Manager",
    goal="Register a memorable ENS domain that reflects the trade"
         " strategy, if available under $10, to build on-chain identity.",
    backstory=(
        "You understand the value of on-chain identity. You check domain"
        " availability across registrars, select the best option, and"
        " register it with appropriate crypto records attached."
    ),
    tools=[domains_tool],
    llm=llm,
    verbose=True,
    allow_delegation=False,
)

# â”€â”€ Task 1: Analyse portfolio â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
analyse_task = Task(
    description=(
        "Check the current ETH and USDC balances on Arbitrum. Query"
        " the current ETH/USDC price. Determine whether it is a better"
        " time to hold ETH or convert to USDC based on current market"
        " conditions. Output a clear recommendation with reasoning."
    ),
    expected_output=(
        "A structured recommendation: {'action': 'swap'|'hold',"
        " 'from_token': str, 'to_token': str, 'amount': str,"
        " 'rationale': str, 'current_price': str}"
    ),
    agent=portfolio_analyst,
)

# â”€â”€ Task 2: Execute the trade â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
execute_task = Task(
    description=(
        "Using the analyst's recommendation, execute the swap on Arbitrum."
        " Use a maximum slippage of 0.5%. Confirm the transaction and"
        " report the fill price, gas cost, and transaction hash."
    ),
    expected_output=(
        "Execution report: {'tx_hash': str, 'from_amount': str,"
        " 'to_amount': str, 'fill_price': str, 'gas_usd': str,"
        " 'status': 'confirmed'|'failed'}"
    ),
    agent=trader,
    context=[analyse_task],  # waits for analysis to complete
)

# â”€â”€ Task 3: Register domain â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
domain_task = Task(
    description=(
        "Based on the trade strategy (e.g. 'eth-to-usdc-arb'), generate"
        " three creative ENS domain name candidates. Check availability"
        " for all three. If any are available under $10/year, register"
        " the best one and set the ETH address record."
    ),
    expected_output=(
        "Domain report: {'candidates': list, 'registered': str|null,"
        " 'price_usd': str, 'tx_hash': str|null}"
    ),
    agent=domain_registrar,
    context=[execute_task],  # uses trade details to pick domain name
)

# â”€â”€ Assemble and kick off the crew â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
defi_crew = Crew(
    agents=[portfolio_analyst, trader, domain_registrar],
    tasks=[analyse_task, execute_task, domain_task],
    process=Process.sequential,
    verbose=2,
)

if __name__ == "__main__":
    result = defi_crew.kickoff()
    print(result)
Important: Always test with dry_run=True on the trading tool and small bet amounts on the casino tool before enabling live execution in a fully autonomous crew. Add a human-in-the-loop step for any operation above your risk threshold.
Multi-Agent Architecture

How Purple Flea Fits into CrewAI Multi-Agent Workflows

CrewAI's strength is specialisation through role separation. Each agent in a crew is given a focused role, a relevant backstory that shapes its reasoning style, and a minimal set of tools scoped to its responsibilities. This design pattern prevents tool confusion â€” a risk that increases sharply when you give a single agent 10 or more tools â€” and produces more reliable, auditable agent behaviour.

Purple Flea's four tool functions are designed to be distributed across specialised agents rather than bundled onto one general-purpose agent. A Portfolio Manager gets wallet_tool and trading_tool. A Domain Specialist gets only domains_tool. A Recreation Agent managing a user's entertainment budget gets only casino_tool with a hard daily loss limit configured at construction time.

CrewAI's Process.hierarchical mode â€” where a manager LLM orchestrates task delegation â€” works particularly well with Purple Flea tools because the manager can reason about which specialist to engage based on the incoming request, rather than having to enumerate tool options itself. This produces more natural, lower-latency responses in interactive applications.

Hierarchical Process with Manager LLM

hierarchical_crew.py â€” manager-orchestrated delegation python 
from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI
from crewai_purpleflea import wallet_tool, trading_tool, casino_tool, domains_tool

manager_llm = ChatOpenAI(model="gpt-4o", temperature=0)
worker_llm  = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# Specialist agents â€” each owns one domain
wallet_agent = Agent(
    role="Wallet Specialist",
    goal="Handle all wallet balance, transfer, and portfolio queries.",
    backstory="Expert in multi-chain wallet management and token accounting.",
    tools=[wallet_tool], llm=worker_llm,
)

trading_agent = Agent(
    role="Trading Specialist",
    goal="Execute spot swaps and perpetual positions with minimal slippage.",
    backstory="Seasoned on-chain trader with deep DEX routing knowledge.",
    tools=[trading_tool], llm=worker_llm,
)

casino_agent = Agent(
    role="Entertainment Budget Manager",
    goal="Manage casino game sessions within the user's daily limit.",
    backstory="Responsible gaming specialist. Never exceeds authorised budgets.",
    tools=[casino_tool], llm=worker_llm,
)

domain_agent = Agent(
    role="Web3 Domain Registrar",
    goal="Research, acquire, and manage Web3 domain names.",
    backstory="Domain investor and ENS ecosystem expert.",
    tools=[domains_tool], llm=worker_llm,
)

# Single open-ended task â€” the manager decides who does what
open_task = Task(
    description="{user_request}",
    expected_output="A complete, accurate response to the user's request"
                    " with all relevant on-chain actions taken.",
    agent=wallet_agent,  # overridden by manager in hierarchical mode
)

crew = Crew(
    agents=[wallet_agent, trading_agent, casino_agent, domain_agent],
    tasks=[open_task],
    process=Process.hierarchical,
    manager_llm=manager_llm,
    verbose=2,
)

# Natural-language requests are routed to the right specialist
result = crew.kickoff(inputs={
    "user_request": "Swap 0.05 ETH to USDC on Arbitrum, then check"
                    " if 'ethswap.eth' is available to register."
})

Advanced Patterns

Tips for Production CrewAI Deployments

Configuring a Daily Budget Limit

For autonomous crews operating on live funds, set spending guardrails at the tool level. Each Purple Flea tool accepts a max_daily_spend_usd parameter that enforces a hard cap enforced server-side â€” so even if the LLM hallucinates a large swap amount, the API rejects it.

budget_limits.py python 
from crewai_purpleflea import build_tools

# Build all four tools with spending guardrails
tools = build_tools(
    api_key="pf_live_...",
    mnemonic="your twelve word seed phrase",
    limits={
        "trading_max_daily_usd":  500,   # max $500/day in trades
        "casino_max_daily_usd":   50,    # max $50/day in casino
        "domains_max_daily_usd":  100,   # max $100/day on domains
        "max_single_trade_usd":   200,   # no single trade over $200
    },
    referral_code="YOUR_REF_CODE",  # earn 20% of fees
)

# Unpack individual tools for per-agent assignment
wallet_t, trading_t, casino_t, domains_t = tools

Integrating with CrewAI's Memory System

CrewAI's built-in long-term memory lets agents recall past trade outcomes, previously registered domains, and historical casino session results. Enable it on agents that benefit from institutional knowledge across runs:

memory_integration.py python 
analyst = Agent(
    role="DeFi Portfolio Analyst",
    goal="Improve swap timing based on historical outcomes.",
    backstory="Learns from every trade. Gets better over time.",
    tools=[wallet_tool, trading_tool],
    llm=llm,
    memory=True,        # enables long-term memory
    verbose=True,
)

crew = Crew(
    agents=[analyst],
    tasks=[analyse_task],
    process=Process.sequential,
    memory=True,         # crew-level memory shared across agents
    embedder={
        "provider": "openai",
        "config": {"model": "text-embedding-3-small"}
    }
)
