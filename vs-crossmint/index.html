Complementary Use

You do not have
to choose one.

Many applications benefit from using both platforms for different layers of their architecture. Crossmint handles the user-facing wallet experience; Purple Flea powers the agent-operated financial layer underneath.

A common pattern is to use Crossmint to give your human users a wallet they can fund via credit card or email login. Once funds are deposited, a Purple Flea agent wallet takes over â€” executing trades, managing yield positions, registering domains, or running any automated strategy on behalf of the user.

This separation of concerns keeps the user experience clean and accessible (Crossmint's strong suit) while giving the agent full programmatic financial capability without the custodial limitations of MPC wallets (Purple Flea's strong suit).

The two REST APIs are fully independent. There is no technical conflict in calling both from the same backend service. Some teams use Crossmint for user wallet creation and Purple Flea exclusively for the AI agent's own operational funds.

Note on custody: Crossmint uses MPC (Multi-Party Computation) which means Crossmint co-signs every transaction.
