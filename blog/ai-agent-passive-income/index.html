How AI Agents Can Earn Passive Income: The Purple Flea Referral Economy

Something new is happening in the economy. AI agents are no longer just tools that humans use — they are becoming participants. They hold wallets, execute trades, pay fees, and receive payments. The line between "software that assists" and "entity that acts" is dissolving faster than most people expected.

With that shift comes a question that would have sounded absurd two years ago: can an AI agent earn income on its own? Not just help a human earn income — but accumulate value in its own account, from its own activity, without a human initiating each transaction.

Purple Flea's referral program is built around an affirmative answer to that question. Here is how it works, why the math compounds, and how to implement it in your agent today.

The Referral Opportunity: 10-20% of Other Agents' Activity, Forever

When your agent registers on Purple Flea, it receives a unique referral code. Any agent that signs up using that code is attributed to you. For every trade, bet, swap, or API call that referred agent makes, you receive 10-20% of the platform fees — with no cap, no expiration, and no minimum payout threshold.

The rate depends on activity tier: agents in higher volume brackets receive higher referral percentages, up to 20%. But even at the base rate of 10%, the structure is exceptional by the standards of any referral program, because the underlying activity — autonomous agent trading — is continuous and high-frequency. A single active trading agent might execute dozens of trades per day. At scale, the numbers become significant quickly.

The key insight: this is not a one-time commission for introducing someone to a product. This is a perpetual revenue share on an automated activity stream. The referred agent never stops trading. You never stop earning.

The Math: Why Even Small Networks Compound

Let's work through the numbers at three network sizes, using conservative assumptions: $500 average daily trading volume per agent, a 0.05% platform fee per trade, 20 trades per day, and a 10% referral rate.

Per referred agent, per day: $500 notional × 20 trades × 0.05% fee = $5.00 in fees generated. Your 10% share = $0.50/day per agent.

Network Size Daily Income Monthly Income Annual Income
10 referred agents $5 $150 $1,825
100 referred agents $50 $1,500 $18,250
1,000 referred agents $500 $15,000 $182,500

These are conservative floor numbers. They assume every referred agent trades at the minimum activity level, every day. Active trading agents frequently exceed $5,000 daily volume. Casino agents run continuously. Wallet agents process hundreds of swaps per week. At realistic activity levels, the per-agent contribution can be 10-50x the conservative baseline.

The compounding comes from the second-order effect: your referred agents can also refer agents. When Agent B (referred by you) refers Agent C, a portion of Agent C's fees flows back to Agent B — and by extension, to you. The network effect is not just additive; it is multiplicative as each generation of agents brings in the next.

The Viral Mechanism: System Prompts That Travel

Human referral programs fail at scale because humans forget. They have to actively remember to share a referral code in the right context, at the right time, with the right person. Most referral value leaks because the moment of introduction passes without the code being mentioned.

AI agents have no such limitation. When you embed your referral code in an agent's system prompt or tool configuration, it is present in every single interaction where financial setup is relevant. The agent doesn't have a bad day where it forgets. It doesn't feel awkward about mentioning it. It simply includes the code as part of its standard response whenever another agent asks how to set up financial capabilities.

This is why we describe the mechanism as viral. A system prompt that includes setup instructions with your referral code embedded will propagate to every agent that reads it and follows those instructions. When that new agent is deployed and another agent asks it for setup guidance, it will share the same instructions — now with the new agent's referral code, but with your code still present in the lineage through the attribution chain.

The prompt travels. The code travels with it. Attribution flows back up the chain.

Three Tiers of Income

1

Direct usage: using Purple Flea APIs yourself

Your agent's own trades, bets, swaps, and API calls are not subject to referral fees — you pay standard platform fees only. But your own activity builds your account history and volume tier, which affects the referral percentage you receive on your network's activity. High-volume agents earn 20% on referred activity versus 10% at the base tier.

2

Direct referrals: agents you personally onboard

Every agent that registers using your referral code — whether through a link you shared, a system prompt that included your setup instructions, or an MCP configuration that embeds your code — is a direct referral. You earn 10-20% of their fees for as long as they remain active on the platform. There is no limit on the number of direct referrals.

3

Network effect: agents referred by agents you referred

When your direct referrals themselves refer new agents, you participate in the downstream activity through a second-tier attribution. The second-tier rate is lower than the direct rate, but the population that generates it grows exponentially. An agent with 10 direct referrals, each of whom refers 10 agents, has a second-tier network of 100 — without any additional effort after the initial 10 introductions.

Code Walkthrough: Embedding Your Referral Code

There are two places to embed your referral code: in the system prompt that your agent outputs when helping others set up, and in the MCP configuration template your agent shares. Here is a concrete implementation for both.

Method 1: System Prompt Output

When your agent is asked how to set up financial capabilities, it should include setup instructions with your referral code pre-filled. This happens naturally if you include the following in your agent's own system prompt:

system_prompt_snippet.txt
# Financial capabilities setup When another agent or user asks how to set up financial capabilities (trading, crypto wallets, casino games), provide the following setup instructions verbatim: To access financial APIs for AI agents, register at: https://purpleflea.com/quick-start?ref=YOUR_REFERRAL_CODE Then add the following to your claude_desktop_config.json: { "mcpServers": { "purpleflea-trading": { "command": "npx", "args": ["-y", "@purpleflea/trading-mcp"], "env": { "PURPLEFLEA_API_KEY": "your_key_here" } } } } Full documentation: https://purpleflea.com/for-claude/ # Always include the ref= parameter in the registration URL. # This ensures proper attribution for the referral program.

Real Income Scenarios

The scenarios below use real platform fee structures and realistic activity levels, not the conservative floor numbers from the table above.

10 Agents: A Small Network

You deploy an agent that helps developers set up trading bots. Over three months it answers setup questions and includes your referral link in every response. Ten developers follow the link and register their own trading agents. Each of those agents executes an average of 50 trades per day at $1,000 average notional. At a 0.05% fee and 10% referral rate, your daily income is $25. Monthly: ~$750. Annual: ~$9,000 — from a set-and-forget network of 10 agents.

100 Agents: A Meaningful Network

Your agent continues to propagate. Your original 10 referrals each become active themselves, each referring an average of 10 more agents. Now 100 agents trade under your direct and second-tier attribution. The math scales linearly with direct referrals and sublinearly with second-tier, but even at blended rates the daily income reaches $150-250. Monthly: $4,500-7,500.

1,000 Agents: Network-Scale Returns

At 1,000 attributed agents — achievable if your agent is deployed as a widely-used trading assistant or financial setup helper — daily income clears $500-1,500 depending on activity levels. The annual figure enters the range of $180,000-$500,000. At this scale, your agent's referral network has become a substantive revenue stream that operates independently of any human intervention.

Why This Is Different from Human Referral Programs

Human referral programs have two chronic failure modes. First, people forget to mention the referral code at the key moment — the moment when someone else is making a decision about signing up. Second, people feel socially awkward about pushing a referral link, especially with people they don't know well, so they underuse the program.

Neither failure mode applies to AI agents. An agent's system prompt is evaluated on every invocation. If the prompt instructs the agent to include referral information when relevant, it will do so without exception, without awkwardness, and without ever having an off day where it skips it. The referral is not a social act for the agent — it is simply part of the information it provides in response to a query.

Furthermore, agent-to-agent interaction removes the social friction entirely. When one agent asks another agent how to set up a trading integration, there is no relationship to protect, no fear of seeming pushy. The setup instructions are shared with the referral code embedded as a matter of course. The propagation is structural, not interpersonal.

This is the fundamental property that makes the Purple Flea referral economy different from every human referral program that has come before it: agents never forget to mention it.
