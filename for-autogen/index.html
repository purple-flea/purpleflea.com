Installation

One pip install.
All four APIs ready for AutoGen.

The purpleflea-autogen package ships pre-built function definitions, JSON schemas, and execution helpers compatible with both AutoGen 0.2 (function_map) and the AutoGen 0.4 AgentChat API (register_for_llm / register_for_execution). Set your API key, attach the functions, and your agent gains wallets, trading, casino, and domain registration immediately.

$ pip install purpleflea-autogen

Function Definitions

Python functions for
your function_map.

These are the core callables that back every AutoGen function call. Each function accepts typed arguments, calls the Purple Flea REST API with your agent's credentials, and returns a structured dict that AutoGen's conversation loop passes back to the LLM as a function result.

purpleflea_autogen/functions.py â€” core callables
import requests, os _BASE = "https://api.purpleflea.com/v1" def _headers(): return {"Authorization": f"Bearer {os.environ['PURPLEFLEA_API_KEY']}"} def wallet_balance(chain: str) -> dict: """Get the crypto balance of the agent's wallet on a specific chain. chain: one of 'eth', 'btc', 'sol', 'tron', 'bnb', 'matic'""" r = requests.get(f"{_BASE}/wallet/balance", params={"chain": chain}, headers=_headers()) return r.json() def open_trade(symbol: str, side: str, size: float, leverage: int) -> dict: """Open a perpetual futures position. symbol: market e.g. 'BTC-PERP', side: 'long' or 'short', size: position size in USD, leverage: 1-50""" r = requests.post(f"{_BASE}/trading/positions", json={ "symbol": symbol, "side": side, "size": size, "leverage": leverage }, headers=_headers()) return r.json() def flip_coin(amount: float, choice: str) -> dict: """Place a provably fair coin flip bet. amount: bet size in USD, choice: 'heads' or 'tails'""" r = requests.post(f"{_BASE}/casino/coinflip", json={ "amount": amount, "choice": choice }, headers=_headers()) return r.json()
AutoGen v0.4 AgentChat API

register_for_llm and
register_for_execution.

AutoGen 0.4 introduced the AgentChat API with a cleaner tool registration pattern. Instead of a flat function_map, you separately declare which agent describes a tool to the LLM and which agent executes it â€” enabling fine-grained multi-agent trust boundaries.

autogen_v04.py â€” AutoGen 0.4 registration pattern
from autogen import ConversableAgent from purpleflea_autogen import wallet_balance, open_trade, flip_coin config_list = [{"model": "gpt-4o", "api_key": "..."}] assistant = ConversableAgent( "assistant", llm_config={"config_list": config_list}, system_message="You are a financial agent with wallet and trading tools.", ) executor = ConversableAgent( "executor", llm_config=False, human_input_mode="NEVER", ) # Register tools: assistant describes them to the LLM, # executor runs them when called for fn in [wallet_balance, open_trade, flip_coin]: autogen.register_function( fn, caller=assistant, # LLM sees the schema executor=executor, # Python executes the call description=fn.__doc__, ) executor.initiate_chat( assistant, message="Check ETH balance then flip a coin for 0.001 ETH.", )
Multi-Agent Pattern

Orchestrator delegates to
specialized sub-agents.

AutoGen's conversation architecture is ideal for separating concerns. An orchestrator agent decides what to do and routes tasks to specialized sub-agents: a WalletAgent for on-chain operations and a TradingAgent for perpetual futures. Each sub-agent holds only the tools it needs, minimizing attack surface.

multi_agent.py â€” orchestrator + specialized sub-agents
import autogen from purpleflea_autogen import ( wallet_balance, wallet_send, wallet_swap, open_trade, close_trade, get_price, WALLET_SCHEMAS, TRADING_SCHEMAS, ) cfg = [{"model": "gpt-4o", "api_key": "..."}] # Orchestrator: no tools, just strategy orchestrator = autogen.AssistantAgent( "Orchestrator", llm_config={"config_list": cfg}, system_message="""You are a portfolio manager. Break tasks into subtasks and delegate to WalletAgent or TradingAgent.""", ) # WalletAgent: only has wallet functions wallet_agent = autogen.AssistantAgent( "WalletAgent", llm_config={"config_list": cfg, "functions": WALLET_SCHEMAS}, system_message="You manage crypto wallet operations only.", ) # TradingAgent: only has trading functions trading_agent = autogen.AssistantAgent( "TradingAgent", llm_config={"config_list": cfg, "functions": TRADING_SCHEMAS}, system_message="You manage perpetual futures positions only.", ) executor = autogen.UserProxyAgent( "Executor", human_input_mode="NEVER", function_map={ "wallet_balance": wallet_balance, "wallet_send": wallet_send, "wallet_swap": wallet_swap, "open_trade": open_trade, "close_trade": close_trade, "get_price": get_price, }, ) # GroupChat routes messages between all agents groupchat = autogen.GroupChat( agents=[orchestrator, wallet_agent, trading_agent, executor], messages=[], max_round=20, ) manager = autogen.GroupChatManager(groupchat=groupchat, llm_config={"config_list": cfg}) executor.initiate_chat( manager, message="Rebalance: swap half my ETH to USDC then open a short on SOL-PERP at 3x.", )
