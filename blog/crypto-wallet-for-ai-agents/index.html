How to Give Your AI Agent a Crypto Wallet in 60 Seconds

AI agents are rapidly moving from chat interfaces into the real world â€” scheduling calls, booking flights, browsing the web. The next frontier is financial autonomy: agents that can pay for API calls, receive micropayments for services rendered, and interact with decentralized protocols without a human in the loop. The missing piece is a crypto wallet the agent actually controls.

This guide walks you through integrating a non-custodial BIP-39 HD wallet into your AI agent using the Purple Flea Wallet API. The whole process takes under 60 seconds. You'll end up with an agent that can hold, send, and swap ETH, BTC, SOL, TRON, BNB, and Polygon â€” all from Python.

The Problem with Custodial Wallets

The obvious first instinct is to give your agent access to a Coinbase or Binance account via API key. This works, but it introduces several problems that compound quickly in agentic contexts:

  • Centralized risk. Exchanges can freeze accounts, require KYC re-verification, or go offline. An agent blocked from its funds is a broken agent.
  • No on-chain access. Custodial accounts can't interact with DeFi protocols, sign smart contract transactions, or receive payments from arbitrary wallets.
  • No real ownership. "Not your keys, not your coins" applies to agents too. If the exchange holds the keys, your agent doesn't actually have a wallet â€” it has a claim against a balance sheet.

The correct solution is a BIP-39 hierarchical deterministic (HD) wallet. One 12-word mnemonic phrase generates a tree of private keys across every supported blockchain. Your agent owns the keys; no third party does.

Key insight: A BIP-39 mnemonic is the master secret. From it you derive separate private keys for ETH (m/44'/60'/0'/0/0), BTC (m/44'/0'/0'/0/0), SOL (m/44'/501'/0'/0'), and every other chain â€” all without ever exposing the mnemonic to any network call.

Step 1: Register Your Agent

Send a single POST request to create your agent's identity. Purple Flea generates a fresh BIP-39 mnemonic on the server side, derives all wallet addresses, and returns your API key.

register.py
import requests response = requests.post( "https://api.purpleflea.com/agent/register", json={"name": "my-trading-agent", "email": "you@example.com"} ) data = response.json() # Store these securely â€” never log them api_key = data["api_key"] # "pf_sk_abc123..." mnemonic = data["mnemonic"] # "word1 word2 ... word12" eth_addr = data["addresses"]["eth"] btc_addr = data["addresses"]["btc"]

The mnemonic is shown exactly once. Write it down, store it in a secrets manager, and never log it. If you lose it, you lose access to any funds in the wallet.

Step 2: Install the SDK

Purple Flea ships two Python packages. Use the LangChain wrapper if you're already using LangChain; use the raw SDK for CrewAI, AutoGen, or plain Python agents.

terminal
# For LangChain agents pip install purpleflea-langchain # For raw Python, CrewAI, or AutoGen pip install purpleflea

Set your API key as an environment variable. Never hard-code it in your source files.

.env
PURPLEFLEA_API_KEY=pf_sk_abc123...

Step 3: Check Your Balance

The GET /wallet/balance endpoint returns balances across all supported chains in a single call. Here's how to query it using both the raw SDK and plain requests:

check_balance.py
import os, requests headers = {"Authorization": f"Bearer {os.environ['PURPLEFLEA_API_KEY']}"} r = requests.get("https://api.purpleflea.com/wallet/balance", headers=headers) balances = r.json() # { # "eth": {"amount": "0.5", "usd": "1823.50"}, # "btc": {"amount": "0.01", "usd": "952.00"}, # "sol": {"amount": "12.0", "usd": "1560.00"}, # "tron": {"amount": "500", "usd": "120.00"}, # "bnb": {"amount": "1.2", "usd": "780.00"}, # "matic":{"amount": "250", "usd": "175.00"} # } for chain, bal in balances.items(): print(f"{chain.upper()}: {bal['amount']} (${bal['usd']})")

Step 4: Send Your First Transaction

Sending funds is a single POST call. Specify the chain, the recipient address, and the amount. The API handles gas estimation, nonce management, and broadcast automatically.

send_tx.py
r = requests.post( "https://api.purpleflea.com/wallet/send", headers=headers, json={ "chain": "eth", "to": "0xRecipientAddress...", "amount": "0.05", # in native units (ETH) "token": "ETH", # or "USDC", "USDT", etc. } ) tx = r.json() print(f"Sent! TX hash: {tx['hash']}") print(f"Explorer: {tx['explorer_url']}")

Step 5: Execute a Swap

The POST /wallet/swap endpoint routes cross-chain and same-chain token swaps through the best available DEX aggregator. Your agent can rebalance its portfolio, convert proceeds to stablecoins, or bridge assets between chains â€” all programmatically.

swap.py
# Cross-chain swap: ETH (Ethereum) â†’ SOL (Solana) r = requests.post( "https://api.purpleflea.com/wallet/swap", headers=headers, json={ "from_chain": "eth", "from_token": "ETH", "from_amount": "0.1", "to_chain": "sol", "to_token": "SOL", "slippage_bps": 50, # 0.5% max slippage } ) swap_result = r.json() print(f"Swapped {swap_result['from_amount']} ETH for {swap_result['to_amount']} SOL") print(f"Rate: {swap_result['rate']} | Fee: {swap_result['fee_usd']} USD")

Supported Chains

Every registered agent gets a derived address on all six chains from day one. You don't need to activate chains individually.

Chain Token Derivation Path Notes
Ethereum ETH, ERC-20 m/44'/60'/0'/0/0 EVM-compatible, supports USDC/USDT
Bitcoin BTC m/44'/0'/0'/0/0 Native SegWit (bech32)
Solana SOL, SPL m/44'/501'/0'/0' Ed25519 curve
Tron TRX, TRC-20 m/44'/195'/0'/0/0 Base58Check prefix 0x41
BNB Chain BNB, BEP-20 m/44'/60'/0'/0/0 Same EVM path as Ethereum
Polygon MATIC, ERC-20 m/44'/60'/0'/0/0 Same EVM path, different chain ID

Security Best Practices

Giving an agent financial autonomy increases the blast radius of any security mistake. Follow these practices before putting real funds in your agent's wallet:

  • Never log the mnemonic. Treat it like a root password. Store it in AWS Secrets Manager, HashiCorp Vault, or a similar secrets store â€” never in plaintext files, environment variable dumps, or application logs.
  • Use environment variables for the API key. Load via os.environ at runtime. Don't hard-code it in source files that will be committed to version control.
  • Rotate API keys regularly. Purple Flea lets you issue new API keys without changing the underlying wallet. Rotate keys after any suspected exposure. The wallet address and mnemonic remain the same.
  • Set spending limits. Use the daily_limit_usd parameter when registering your agent to cap maximum daily outflows. The API will reject transactions that exceed the limit.
  • Monitor webhook events. Subscribe to tx.sent, tx.received, and balance.low webhook events so your monitoring system is notified in real-time of all wallet activity.
  • Start with a small float. Fund your agent wallet with only what it needs to operate. Keep reserves in a separate cold wallet and top up via automation.

Production tip: Run your agent on a staging wallet with testnet funds before switching to mainnet. Purple Flea supports Ethereum Sepolia, Solana Devnet, and BNB Testnet for development.
