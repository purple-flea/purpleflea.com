Overview

Why Purple Flea + LangChain?

LangChain's tool-calling pattern is the fastest way to give language models agency over real-world systems. Purple Flea extends that pattern into decentralised finance, provably-fair gaming, and the on-chain identity layer â€” domains that most tool libraries completely ignore.

Every BaseTool in purpleflea-langchain is built around a strict Pydantic input schema, which means the LLM always receives structured, validated arguments â€” no silent misparses, no truncated addresses, no garbled amounts. The underlying Purple Flea API is entirely non-custodial: private keys are derived client-side using BIP-32/BIP-44, signed locally, and broadcast directly to the relevant network. Purple Flea never touches your private keys.

The package wraps four distinct API surface areas into a coherent, idiomatic LangChain interface. You can pass all four tools to a single create_tool_calling_agent call and let the model decide which capability to invoke based on user intent, or you can use the tools independently in chains and pipelines.

Whether you are building an autonomous DeFi portfolio manager, a multi-chain arbitrage bot, a gaming assistant that manages bankroll on behalf of a user, or a domain acquisition agent that snaps up expiring Web3 names, Purple Flea for LangChain gives you the foundation to ship it fast â€” with zero compromise on security or correctness.

Installation

Get Running in Three Steps

  • Install the package from PyPI using pip, poetry, or uv. Python 3.10+ required.
  • Set your Purple Flea API key as the environment variable PURPLEFLEA_API_KEY. Get one free at purpleflea.io.
  • Import the four tools, instantiate them with your config, and pass them to any LangChain agent.
terminal bash 
# Install with pip
pip install purpleflea-langchain

# Or with poetry
poetry add purpleflea-langchain

# Or with uv
uv add purpleflea-langchain

# Set your API key (get one free at purpleflea.io)
export PURPLEFLEA_API_KEY="pf_live_your_key_here"
export OPENAI_API_KEY="sk-..."  # or your preferred LLM provider
Quick Start

Full Agent Example with All Four Tools

The following example creates a complete tool-calling agent that can manage a crypto wallet, execute trades, play casino games, and register domains â€” all driven by natural language.

agent.py â€” full tool-calling agent python 
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from purpleflea_langchain import (
    PurpleFlecaCasinoTool,
    PurpleFlecaTradingTool,
    PurpleFlecaWalletTool,
    PurpleFlecaDomainsTool,
)

# â”€â”€ 1. Instantiate the four tools â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
casino_tool   = PurpleFlecaCasinoTool(
    api_key="pf_live_your_key",
    chain="ethereum",
    default_bet_usd=5.0,
)

trading_tool  = PurpleFlecaTradingTool(
    api_key="pf_live_your_key",
    default_chain="arbitrum",
    slippage_bps=30,
)

wallet_tool   = PurpleFlecaWalletTool(
    api_key="pf_live_your_key",
    mnemonic="your twelve word seed phrase here",  # never leaves your machine
    default_chain="ethereum",
)

domains_tool  = PurpleFlecaDomainsTool(
    api_key="pf_live_your_key",
    registrar="ens",  # or "unstoppable", "handshake"
)

tools = [casino_tool, trading_tool, wallet_tool, domains_tool]

# â”€â”€ 2. Build the prompt â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
prompt = ChatPromptTemplate.from_messages([
    ("system", (
        "You are a DeFi agent powered by Purple Flea. You can manage"
        " crypto wallets, execute trades, play provably-fair casino games,"
        " and register Web3 domains. Always confirm destructive actions"
        " with the user before executing."
    )),
    MessagesPlaceholder("chat_history", optional=True),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])

# â”€â”€ 3. Create the agent â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
llm   = ChatOpenAI(model="gpt-4o", temperature=0)
agent = create_tool_calling_agent(llm, tools, prompt)

executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=10,
    handle_parsing_errors=True,
)

# â”€â”€ 4. Run it â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
if __name__ == "__main__":
    result = executor.invoke({
        "input": (
            "Check my ETH balance, then swap 0.01 ETH for USDC on Arbitrum,"
            " and register 'myagent.eth' if it's available."
        )
    })
    print(result["output"])
Note: The mnemonic you pass to PurpleFlecaWalletTool is used exclusively for local key derivation. It is never transmitted to Purple Flea servers. All transaction signing happens in-process using the eth_account library before broadcast.
Individual Tool Usage

Tool-by-Tool Code Examples

Each tool can also be invoked directly in a standalone chain or tested in isolation. Here are targeted examples for each of the four tool capabilities.

Wallet: Check Balance Across Chains

The wallet tool exposes balance queries for ETH, ERC-20 tokens, BTC, SOL, and MATIC in a single call. You can specify a chain, token contract address, or leave both blank to get a full portfolio overview in USD.

wallet_balance.py python 
import asyncio
from purpleflea_langchain import PurpleFlecaWalletTool

wallet = PurpleFlecaWalletTool(
    api_key="pf_live_...",
    mnemonic="your seed phrase",
)

# Synchronous invocation (LangChain handles the async internally)
balance = wallet.invoke({
    "action":  "get_balance",
    "chain":   "ethereum",
    "token":   "ETH",         # omit for all tokens
    "address_index": 0,         # BIP-44 address index
})

# Returns a JSON-serialisable dict:
# {"address": "0xABC...", "token": "ETH", "balance": "1.2340",
#  "balance_usd": "4532.11", "chain": "ethereum"}
print(balance)

# Async usage with arun
async def check_portfolio():
    result = await wallet.arun({
        "action": "get_portfolio",
        "chains": ["ethereum", "arbitrum", "polygon"],
    })
    return result

asyncio.run(check_portfolio())

Casino: Provably-Fair Coin Flip

Every casino game uses a Verifiable Random Function (VRF) seed committed on-chain before the result is revealed. Your agent can query the VRF proof and verify the outcome independently using the included verify_outcome() helper.

casino_example.py python 
from purpleflea_langchain import PurpleFlecaCasinoTool
from purpleflea_langchain.utils import verify_outcome

casino = PurpleFlecaCasinoTool(
    api_key="pf_live_...",
    chain="polygon",     # low-fee chain recommended for small bets
    default_bet_usd=2.0,
)

# â”€â”€ Coin flip â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
flip = casino.invoke({
    "game":   "coin_flip",
    "choice": "heads",
    "bet_usd": 5.0,
})
print(flip)
# {"game": "coin_flip", "choice": "heads", "result": "heads",
#  "outcome": "win", "payout_usd": "9.75", "vrf_seed": "0xdef...",
#  "tx_hash": "0x123...", "chain": "polygon"}

# â”€â”€ Verify the outcome is provably fair â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
is_valid = verify_outcome(
    vrf_seed=flip["vrf_seed"],
    result=flip["result"],
    game="coin_flip",
)
print(f"Outcome verifiable on-chain: {is_valid}")

# â”€â”€ Dice roll â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
dice = casino.invoke({
    "game":        "dice_roll",
    "target":      "over",
    "target_value": 50,
    "bet_usd":     10.0,
})
print(dice)
