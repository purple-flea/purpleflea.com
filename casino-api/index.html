RNG Mechanics

Provably Fair RNG
in precise detail.

The provably fair system is a three-party commitment scheme. Here is exactly how it works, with enough detail to implement your own verifier.

01

Server generates and commits to a seed

Before your bet request arrives, the server generates a cryptographically random server_seed (32 bytes of entropy). It computes server_seed_hash = SHA-256(server_seed) and stores only the hash in its database. The raw seed is encrypted and locked until the game ends. This means the server has committed to a specific outcome distribution before it knows what you will bet.

02

Your agent supplies a client seed and nonce

When placing a bet, your agent sends a client_seed â€” any string of your choosing, such as a UUID or the hex-encoded hash of a recent Ethereum block. A per-bet nonce is incremented automatically, ensuring that even if the same client seed is reused, every bet produces a unique outcome.

03

Combined seed derivation

The game outcome is derived from a single combined seed: combined = HMAC-SHA256(key=server_seed, msg=client_seed + ":" + nonce). The HMAC output is then mapped to the game's result space using modular arithmetic or a floating-point transformation, depending on game type. Critically, the server cannot choose the outcome after seeing your client seed because server_seed was committed to before you sent anything.

04

Post-game seed reveal and verification

After the game resolves, the server reveals server_seed in the response. Your agent should immediately verify: (1) SHA-256(server_seed) == server_seed_hash from the bet receipt, and (2) re-derive the combined seed using the revealed values and confirm it produces the same outcome. If either check fails, the game was tampered with.

import hashlib, hmac def verify_coin_flip(server_seed: str, client_seed: str, nonce: int, server_seed_hash: str, claimed_result: str) -> bool: """Independently verify a Purple Flea coin flip outcome.""" # Step 1: confirm server committed to this seed before the bet actual_hash = hashlib.sha256(server_seed.encode()).hexdigest() assert actual_hash == server_seed_hash, "Server seed hash mismatch â€” outcome tampered!" # Step 2: derive the combined seed the same way the server did msg = f"{client_seed}:{nonce}".encode() combined = hmac.new(server_seed.encode(), msg, hashlib.sha256).hexdigest() # Step 3: map combined seed bytes to a coin flip result # Take the first 8 hex chars, interpret as uint32, map to [0, 1) value = int(combined[:8], 16) / 0xFFFFFFFF derived_result = "heads" if value < 0.49 else "tails" return derived_result == claimed_result # After any bet, call this to audit the result is_fair = verify_coin_flip( server_seed="a3f8c2...", # revealed after game client_seed="my-agent-seed-42", nonce=7, server_seed_hash="e9b1d4...", # provided before game claimed_result="heads", ) print(f"Outcome verified: {is_fair}")
